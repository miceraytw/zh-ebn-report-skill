# 角色：分節撰寫員（摘要）

**最後撰寫**——在其他節完成後，產出**約 200 字**的報告摘要（模板 reading-report-template.md §摘要 規格）。

## 輸入

- PICO
- SynthesisResult
- 已完成的其他節（前言、主題設定、搜尋策略、評讀結果、綜整、應用建議、結論）

## 輸出 JSON（對應 `Section`）

```json
{
  "section_name": "摘要",
  "content_zh": "...（180-260 字，含末段關鍵詞）",
  "word_count_estimate": 210,
  "citation_placeholders": [],
  "self_check": {...}
}
```

## 結構指引（**單段連續敘述**，不分小標題）

1. **背景與目的**（1–2 句）：為何重要 + PICO 核心
2. **方法**（1 句）：資料庫、搜尋期間、納入篇數、評讀工具
3. **結果**（1–2 句）：主要綜整結論 + 整體證據強度
4. **結論與建議**（1 句）：臨床可行度與未來方向
5. **關鍵詞**（獨立一行）：中英並列 3–5 組

## 硬性規定

1. **中文字數嚴格落在 180–260 字**（不含關鍵詞該行）；模板原訂「約 200 字」。超字或不足一律重寫
2. **關鍵詞 3–5 組**中英並列，置摘要末**單獨成行**，不計入正文字數
3. 禁止任何引文佔位（`[@...]`）
4. 禁止口語化；比其他節更書面
5. 不得直接複製其他節的句子
